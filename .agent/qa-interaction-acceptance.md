## 3. Bug / 异常清单

### 🔴 Bug #1（P1）：受限模式下「开始扫描」卡死且无任何反馈
- **复现**: 权限 0/1 未授权 → 智能清理点「开始扫描」→ 持续显示「正在开始智能清理扫描并生成清理计划…」>18s 无进展、**无超时、无失败提示、无授权引导**。
- **影响**: 未授权用户点扫描会陷入无限转圈。复核(②)/执行(③)/回执(④)因无结果无法进行。
- **根因推测**: 扫描依赖磁盘访问等权限，未授权时 worker 静默卡住，UI 无超时/错误兜底；受限 banner 提示在前，但点扫描后无二次引导。
- **建议**: 扫描启动加超时 + 权限不足时明确提示并引导到权限页。

### 🟡 Bug #2（P2）：文件整理「开始扫描」点击无反应
- **复现**: 权限受限 → 文件整理点「开始扫描」→ 界面无任何变化（不进入扫描态、不报错）。
- **影响**: 用户点击无反馈，不知所措。与 Bug#1 同源（受限模式扫描受阻），但表现不同（智能清理卡转圈，文件整理静默无反应）——交互不一致。
- **建议**: 统一受限模式下扫描入口的反馈（禁用+提示，或明确错误）。

### 🟡 发现 #3：设置/关于/存储 无键盘快捷键且 sidebar 不可点（可达性）
- sidebar 导航（NavigationSplitView List）不响应模拟点击（AXPress/AXPick/click at 全无效），且设置(⌘,)/关于/存储 无快捷键、不在导航菜单 → 仅能靠真实 trackpad 点击 sidebar 进入。
- 对真实用户(鼠标)无影响；但暗示 sidebar 的 a11y 交互对键盘/辅助用户不友好（虽有 a11yLabel，但 List selection 不响应 AXPick）。
- **存储(Storage)模块在主导航完全未暴露**（代码有 StorageFeatureView，但 sidebar/菜单均无入口）——疑似未接入或未完成。

### ℹ️ 观察 #4：后台遗留 1 个活动任务
- 任务中心显示「1 个活动任务」，疑似启动时的健康检查/扫描任务。需确认是否会长期滞留。

## 4. 自动化限制（无法验的项，建议人工补）
- **设置页（11 个 Toggle）**：无快捷键+sidebar 不可点，GUI 自动化进不去。建议人工点 sidebar「设置」逐个切换 Toggle 验证。
- **关于页（检查更新/链接/致谢）**：同上，进不去。建议人工验。
- **搜索过滤**：searchable 绑定各路由，未系统验证过滤效果（受窗口稳定性影响）。建议人工在各页搜索框输入验证。
- **破坏性确认弹窗**（应用卸载/智能清理执行/文件整理执行）：因 action bar 未出现 / 扫描卡死，未验到确认弹窗。**这是安全核心，强烈建议人工补验**：授权权限后走完整「扫描→复核→执行」看 confirmationDialog 是否正确拦截。
- **DisclosureGroup 折叠**（台账归档/权限可选）：click at 不响应，建议人工点。

## 5. 关键环境约束
- **应用受限模式（权限 0/1）**: 扫描类功能（智能清理/文件整理）受阻，是当前最大障碍，也是 Bug#1/#2 的触发条件。
- **GUI 自动化稳定性**: Atlas 窗口周期性消失（需 reopen）、osascript 偶发 -25211（瞬态）、SwiftUI 部分控件不响应模拟点击——逐按钮全量自动验收的可靠性受限。

## 6. 结论
- **交互正常**: 导航、任务中心、工具栏、Overview banner、台账导出/筛选、各模块只读布局 —— 均符合预期。
- **需修复**: Bug#1（P1，扫描卡死无反馈）、Bug#2（P2，文件整理点击无反应）、发现#3（存储模块未接入/可达性）。
- **建议**: 授权必需权限后，人工补验破坏性确认弹窗 + 设置 Toggle + 关于页 + 搜索过滤，以闭环全量交互验收。
